# be2 Docker Guide

Этот проект содержит примеры бэкенд-сервиса на Go. Ниже — краткая инструкция, как собрать и запустить его в Docker вместе с базой данных PostgreSQL.

## Структура

Все файлы, относящиеся к запуску через Docker, лежат в каталоге [`src/`](src):

- [`Dockerfile`](src/Dockerfile) — многоступенчатая сборка бинаря `cmd/server`.
- [`docker-compose.yml`](src/docker-compose.yml) — поднимает приложение и PostgreSQL.
- [`Makefile`](src/Makefile) — содержит вспомогательные команды.

## Предварительные требования

- Docker Engine 24+
- Docker Compose (поставляется вместе с Docker Desktop или `docker compose` в CLI)

## Сборка и запуск

1. Перейдите в каталог `src` (это важно, потому что `docker-compose.yml` лежит именно здесь):

   ```bash
   cd src
   ```

2. Соберите и поднимите контейнеры:

   ```bash
   docker compose up -d --build
   ```

   Команда соберёт образ приложения из [`Dockerfile`](src/Dockerfile), запустит базу `postgres:16-alpine` и приложение. Приложение доступно на `http://localhost:8080`, база — на порту `5432`.

3. Чтобы посмотреть логи:

   ```bash
   docker compose logs -f app
   ```

4. Остановить и удалить контейнеры и тома можно так:

   ```bash
   docker compose down -v
   ```

## Использование Makefile

Находясь в каталоге `src`, можно пользоваться командами `make` (они просто обёртки над стандартными командами):

- `make up` — то же, что `docker compose up -d --build`.
- `make down` — останавливает и чистит контейнеры.
- `make build`, `make test` и др. — локальная сборка и тесты без Docker.

## Частые проблемы

- **Docker не видит Dockerfile.** Убедитесь, что файл называется `Dockerfile` (без расширения) и вы запускаете команды из каталога `src`.
- **Ошибка `unknown Go version`.** В образе используется Go `1.22.4`. Убедитесь, что у вас свежая версия Docker, чтобы он смог скачать этот образ. Версия в [`go.mod`](src/go.mod) также обновлена.
- **Приложение завершается сразу после запуска.** Проверьте логи контейнера `app`: `docker compose logs -f app`.

Если нужно изменить конфигурацию базы или приложения, редактируйте переменные окружения в [`docker-compose.yml`](src/docker-compose.yml).
